# Treemap Hierárquico com Drill-down e Cores por Status

## Objetivo
Reescrever o componente `ChartTreemap` para suportar:
- **Drill-down**: clique para navegar entre níveis (Área → Setor → Conjunto → Serviço)
- **Cores por Status**: Vermelho (Pendente), Amarelo (Em Andamento), Verde (Finalizado)
- **Raiz fixa**: "Cabo de Santo Agostinho"
- **Rótulos completos**: caminho hierárquico + contagem

## Requisitos do usuário
- Raiz: Cabo de Santo Agostinho
- Níveis: Área > Setor > Conjunto > Serviço
- Tamanho: contagem de relatórios
- Cores: Pendente=Vermelho, Em Andamento=Amarelo, Finalizado=Verde
- Drill-down com clique
- Rótulos com caminho completo + quantidade

## Implementação

### 1. Atualizar `HierarchicalNode` em `src/lib/chart-utils.ts`

Adicionar campos de metadata:
```ts
export interface HierarchicalNode {
  name: string;
  children?: HierarchicalNode[];
  size?: number;
  status?: string;        // Status predominante do grupo
  totalByStatus?: Record<string, number>; // contagem por status
}
```

### 2. Reescrever `aggregateHierarchy` em `src/lib/chart-utils.ts`

A nova versão deve:
- Aceitar um parâmetro opcional `statusKey` (campo para colorir)
- Preservar `totalByStatus` em cada nó (contagem de cada status nos filhos)
- Calcular `status` como o status predominante (maior contagem)
- No nível folha: `size` = contagem total, `status` = status da maioria

### 3. Reescrever `src/components/chart-treemap.tsx`

**Drill-down com React state:**
- `currentPath: string[]` — array de nomes do caminho atual (ex: `["Refrigeração", "Compressor X"]`)
- `currentData: HierarchicalNode` — nó raiz atual baseado no path
- Função `drillInto(nodeName)` — adiciona ao path, atualiza currentData
- Função `drillUp()` — remove último nível do path
- Breadcrumb navigation no topo do treemap

**Cores por Status:**
- Mapa de cores:
  - `Pendente` / `Aberto` / `Não Iniciado` → `#EF4444` (vermelho)
  - `Em Andamento` / `Em execução` / `Programada` → `#F59E0B` (amarelo)
  - `Finalizada` / `Concluído` / `Fechado` → `#10B981` (verde)
  - Default → `#93C5D8` (cinza)
- Usar `STATUS_COLORS` existente como base, com fallback

**CustomContent renderer:**
- Retângulo com cor baseada no `status` do nó
- Label: nome do nó + contagem (ex: "Refrigeração (23)")
- Fonte adaptativa ao tamanho do tile
- Tiles < 30x20 ocultos

**Tooltip:**
- Caminho completo (ex: "Refrigeração / Compressor X / Conjunto Y")
- Contagem total
- Breakdown por status (se disponível)

**Breadcrumb:**
- "Cabo de Santo Agostinho > Área > Setor > ..."
- Clique em qualquer nível do breadcrumb navega até ele
- Estilo: texto链接 azul claro, separador ">"

### 4. Atualizar integração em `src/routes/_app.preditivas.tsx`

- Chamar `aggregateHierarchy` com `statusKey: "Status"`
- Passar dados para `ChartTreemap`
